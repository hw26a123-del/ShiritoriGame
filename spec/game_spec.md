# しりとりゲーム仕様書「爆速しりとり」

## 1. 概要
制限時間内にタイピングでしりとりを続ける、PC向けのゲーム。タイトルは「爆速しりとり」。

## 2. 画面レイアウト（UI構成）
ゲームプレイ中は以下のレイアウトで構成される：
- **画面上部中央**: 最初の単語、およびしりとりで繋がっている現在の単語（ひらがな）を表示。
  - **ひらがな単語の下部**: 少し小さく、その単語のローマ字表記を表示。しりとりを繋げるための最後の文字（判定対象文字）に該当するローマ字の部分（例：「らくご」の「go」など）のみ、赤色で強調表示する。
- **画面中部中央**: プレイヤーが文字を入力するテキスト入力欄（小文字のローマ字のみを許可）。
- **画面上部右**: タイマー（残り時間数値）を表示。
- **画面上部左**: スコアを表示。
- **画面下部**: タイマーの残り時間を示す進捗バー（インジケーター）を表示。

## 3. ゲームの進行状態と画面表示

### 3.1 スタート画面
ゲーム開始前はスタート画面を表示する。
- **画面上部中央**: タイトル「爆速しりとり」を表示。
- **画面中央**: 以下の要素を上から順に表示。
  1. **スタートボタン**
  2. **最高スコア**

### 3.2 ゲームオーバー画面
ゲームオーバー時は以下の要素を上から順に画面中央に表示する：
1. **リスタートボタン**
2. **最終スコア**
3. **最高スコア**

## 4. ゲームルール（コアメカニクス）
- **入力操作**: PC（キーボード）によるタイピング入力。
- **ゲームの進行（システムとの交互プレイ）**:
  - ゲームは「プレイヤーのターン」と「システムのターン」が交互に繰り返される。
  - プレイヤーが正しい単語を入力して確定（Enter）すると、システムが自動的に次に繋がる単語を提示する。
  - プレイヤーは、システムが提示した最新の単語の「最後の文字（赤色のローマ字部分）」から繋がるように、次の単語をローマ字で入力する。
- **入力・確定ルール**:
  - **小文字のローマ字のみ**の入力を対象とする。
  - **ローマ字の綴りルール**:
    - 「si」や「shi」などの複数の綴りパターンをすべて許可する。
    - **表記揺れの完全互換サポート**: プレイヤーが入力しやすいように、代表的なローマ字の綴り揺れ（「ふ」に対する「fu」と「hu」、「し」に対する「shi」と「si」、「ち」に対する「chi」と「ti」、「つ」に対する「tsu」と「tu」、「じ」に対する「ji」と「zi」など）を完全に互換・許容します。例えば、システム側が「koppu」などの単語を提示し、次の文字が「ふ」の時、プレイヤーが「hu」と入力しても「fu」から始まるものとしてしりとり判定を通過させます。
    - 小書き文字（「っ」や「ょ」など）の入力規則について、「xtu」「ltu」「xyo」「lyo」のように「x」や「l」を付与して単体入力するタイプと、「tta」「sha」などのように前後の子音や結合文字と組み合わせるタイプの両方をすべて許可する。
  - Enterキーが押されたときに入力を確定し、判定を行う。
  - 誤った入力（しりとりとして繋がっていない、すでに使われた、等）や小文字ローマ字以外の文字が入力された、あるいは辞書判定で存在しないとされた場合は、入力された文字が震えながら消去される。
- **ゲーム開始条件**:
  - ゲーム開始時の最初の単語は、システム側があらかじめ用意した「3文字の単語」を表示する。ただし、この単語の最後の文字に「ん」は使用しない。
- **辞書判定**:
  - 入力されたローマ字をひらがなに逆変換した上で、無料かつ事前登録不要の外部辞書APIを利用して実在する言葉かどうかの判定を行う。
  - **ローカル辞書の強化**: 外部辞書APIでヒットしない日常語（例：「らっこ」など）の漏れを防ぐため、ローカルの静的辞書（言葉リスト）に日常的に多用される言葉を幅広く登録・内蔵して辞書判定を補強する。システム側の解答バリエーションを豊かにし、よりダイナミックなゲーム展開を実現するため、**6文字以上の長単語を100個以上、ローカル辞書に新規搭載**している。これらの長単語は、しりとりが継続できるように「ん」で終わらない単語のみを選定している。
  - **「る」から始まる単語の補強による降参不具合防止**: プレイヤーが「る（ru）」で終わる言葉（例：「かえる」「みる」など）を入力した際、システムが次に繋げる「る」から始まる単語を見つけられずに降参（システムが言葉を思いつけずゲームオーバー）となってしまう不具合を防ぐため、ローカル辞書に「るびー」「るーる」「るすばん」「るーぷ」「るり」「るーぺ」などの一般的な「る」から始まる言葉を多数内蔵・補強している。
  - **「ぬ」から始まる単語の補強による降参不具合防止**: プレイヤーが「いぬ（inu）」などの「ぬ」で終わる言葉を入力した際、同様にシステムが次に繋げる「ぬ」から始まる単語を見つけられずに降参（システムが言葉を思いつけずゲームオーバー）となる不具合を防ぐため、ローカル辞書に「ぬいぐるみ」「ぬりえ」「ぬま」「ぬーどる」「ぬくもり」「ぬかづけ」などの一般的な「ぬ」から始まる言葉を内蔵・補強している。
- **制限時間（タイマー）システム**:
  - **ゲーム開始時、制限時間は10秒。**（スピーディなゲーム性の確保のため）
  - タイマーが0秒（残り時間なし）になった場合は、ゲームオーバーとなる。
  - しりとりを一回成功（完了）させるごとに、次回入力のタイマーの最大時間が1秒ずつ減少。
  - タイマー最大時間の最小値は5秒（5秒未満には減少させない）。
  - **タイマーの一時停止**: プレイヤーがEnterキーを押して入力を確定した瞬間（辞書APIの判定中やシステムの応答待ちなど、読み込みラグの間）は、タイマーのカウントダウンを一時停止（ポーズ）する。システムの次の単語が決定され、次のプレイヤーのターンが開始された（単語が表示された）タイミングで、新しい制限時間でタイマーを再始動する。
  - **タイムバーの追従性最適化とレイアウト崩れ防止**: 制限時間が5秒に近づいた時、あるいは残り時間が少なくなった際の緊迫感を正しくプレイヤーに伝えるため、画面下部のタイムバー（プログレスバー）の縮小アニメーションが実際のタイマー数値（timeLeft）と完全に同期するように調整する。具体的には、更新イージングを等速（linear）に設定し、アニメーション追従の遅延を解消する。また、制限時間が5秒の極限スピード時などに、履歴の表示数増加や回答エラーのメッセージバルーン表示によって画面のメインコンテンツが縦に伸びた際、一番下のタイムバーが縦に押し潰されて表示が小さく（細く）なってしまうレイアウト不具合を防ぐため、フッターおよびヘッダーにはフレックスボックス内の縮小防止設定（flex-shrink: 0 / shrink-0）を施して常に定寸（高さを維持）で描画する設計とする。
- **しりとり判定の特殊ルール**:
  - **濁音・半濁音の緩和（一方通行動作）**: 
    - 前の単語が濁音・半濁音で終わる場合、濁点・半濁点を取り除いた清音から始まる言葉で繋いでもよい。（例：「らくご」の「ご（go）」から「こあら」の「こ（ko）」として繋ぐことは許容する）
    - 前の単語が清音で終わる場合、次の単語を濁音・半濁音で始めることはできない。（例：「らっこ」の「こ（ko）」から「ごりら」の「ご（go）」に繋ぐことは不可）
  - **長音（ー）の処理**: 「るびー」のように最後が長音符「ー」で終わる場合は、その1つ手前の文字（「び（bi）」）を対象として繋ぐ。
  - **小書き文字（拗音・促音）の処理（UI・システム不整合の解消）**:
    - 「きしゃ（kisha）」のように拗音で終わる場合、ローマ字の表記（トークン）は「sha」となります。ひらがなルールのみを適用すると「ゃ」➔「や（ya）」となり不整合が発生するため、**UI上で赤色表示されているローマ字表記（例：「sha」）から直接しりとりを繋ぐこと（例：「shatyou」）を正解とします。**
    - プレイヤーは、画面に赤字で強調表示されているローマ字表記（例：「sha」）から直接しりとりを繋がなければなりません。ひらがな変換後の「ゃ ➔ や（ya）」でしりとりを繋ぐことは不許可（対象外）とし、直感的なローマ字ベースの接続のみを許可します。
    - 「っ」などの促音で終わる単語はしりとり不可（または「つ」として扱う）とする。
  - **重複単語の禁止**: 一度使った単語を再入力した場合は、誤った入力（ローマ字以外やしりとり不成立時と同じ）とみなし、入力文字を震えさせながら消去する。これはシステムが入力した単語、プレイヤーが入力した単語の両方に適用される。
- **しりとりルール・ゲームオーバー条件**:
  - 「ん（n）」で終わる単語を入力した場合は、即座にゲームオーバーとなる。
  - 制限時間が0秒になった場合もゲームオーバーとなる。
- **スコアシステム**:
  - しりとりを一回成功させるごとに、スコアに100点ずつ加算。
